# בריף פרויקט — אייג'נט AI ליצירת מפרטי אימוג'י מותאמי-פלטפורמה

> קובץ זה מסכם את כל ההחלטות שהתקבלו עד כה בשיחת התכנון, ומיועד לשמש כהקשר פתיחה בעבודה על הפרויקט (למשל ב-Claude Code). יש להניח אותו בשורש הריפו.

## 1. דרישת המטלה (תמצית)
קורס: פיתוח מערכות מבוססות LLMs. יש לבנות **Agent ממוקד-משימה** (לא צ'אט כללי) באמצעות **LangGraph** ו-**OpenRouter** כספק מודלים, עם ניטור ב-**LangSmith**.

דרישות ליבה:
- Workflow מובנה בגרף (נודים + קצוות), כולל GuardRails, Gates, HITL, Evaluator.
- לפחות הרחבה אחת: RAG / Web-Search / Code-Exec / MCP.
- ממשק **CLI** אינטראקטיבי (חובה) — Streaming של שלבים, help, טיפול בקבצים (נתיב מקומי), הצגת עלות/שימוש, טיפול בשגיאות. גרסת Desktop (React+Tauri+FastAPI) היא בונוס בלבד ולא פוטרת מדרישות ה-CLI.
- שלושה Evals, כל אחד שומר תוצאות ל-CSV נפרד תחת `evals/`:
  1. השוואת מודלים (זמן/איכות/אמינות, LLM-as-judge לפי rubric קבוע).
  2. בדיקת עלויות (לפחות מודל בתשלום אחד; עלות תיאורטית למודלים חינמיים).
  3. מקצועיות ואי-זליגה (≥5 שאלות מחוץ לתחום, נמדד כ-refusal rate).
- הגשה: ריפו GitHub + `agent.exe` (PyInstaller) תחת Releases, `manifest.json` בשורש, `README.md` מפורט, `.env.example` (ללא ערכים אמיתיים).

## 2. הרעיון שנבחר
אייג'נט שמקבל **תיאור טקסטואלי חופשי** של אימוג'י ופלטפורמת/ות יעד (Apple / Google / Samsung...), ומפיק **מפרט עיצובי מובנה** (צבעים, צורות, קווי מתאר, מאפיינים ייחודיים) המבוסס על מדריכי סגנון אמיתיים (RAG) — לפני שהוא בכלל מגיע לניסוח, הוא עובר שרשרת הבנה: רגש → מחווה → פירוק למאפיינים → בדיקה אם כבר קיים אימוג'י דומה → החלטה אם צריך חדש.

**שלב ליבה (MVP):** תוצר = מפרט טקסטואלי + הסבר החלטות. הכל דרך CLI.
**שלב בונוס:** הפקת קובץ SVG בפועל על סמך המפרט (Code-Exec), עם אישור HITL לפני יצירת קובץ.

רקע: המשתמשת ביצעה בעבר פרויקט ML שבו אימנה מודל שמזהה מאיזו פלטפורמה מקורי אימוג'י נתון (על סמך תמונה). כוונה לשלב מודל זה **בשלב הבונוס בלבד**, בתוך ה-Evaluator: להריץ אותו על ה-SVG שנוצר ולבדוק אם הוא מזהה אותו כשייך לפלטפורמת היעד. **תלוי בבדיקת היתכנות** — המודל אומן על תמונות אמיתיות, לא ברור שיתמודד טוב עם SVG פשוט שנוצר בקוד. אם לא יעבוד טוב — ממשיכים בלי זה, ה-Evaluator מבוסס LLM-as-judge בלבד עונה במלואו על הדרישה.

## 3. ארכיטקטורת הגרף (LangGraph)

| # | נוד | תפקיד |
|---|---|---|
| 0 | Guardrail | בדיקה שהבקשה בתחום עיצוב אימוג'י; חוסם בקשות מחוץ לתפקיד |
| 1 | הבנת רגש | LLM: זיהוי הרגש המרכזי בתיאור |
| 2 | זיהוי מחווה | LLM: חילוץ הפעולה/מחווה הפיזית |
| 3 | פירוק למאפיינים | LLM עם פלט JSON: רכיבי פנים/גוף נפרדים |
| 4 | בדיקת דמיון קיים | Retrieval: השוואה מול מאגר אימוג'ים קיימים (unicode + תיאורים) |
| 5 | **Gate** — האם צריך חדש? | קצה מותנה: אם נמצאה התאמה קרובה → תשובה ישירה ודילוג על שאר הגרף; אחרת → המשך |
| 6 | RAG + התאמה לפלטפורמה | שליפת מדריך סגנון לכל פלטפורמת יעד + ניסוח מפרט |
| 7 | הסבר החלטות | LLM מנמק את הבחירות תוך הפניה למקורות |
| — | Evaluator | ציון לפי rubric (רלוונטיות, דיוק, שלמות, פורמט); ציון נמוך → חזרה לנוד 6 |
| 8 (בונוס) | Code-Exec | יצירת קובץ SVG בפועל, לאחר Gate/HITL |
| — | Output | הצגת מפרט (+ בבונוס: נתיב קובץ), עלות, זמן |

זרימה: `קלט → Guardrail → הבנת רגש → זיהוי מחווה → פירוק למאפיינים → בדיקת דמיון קיים → Gate → [RAG+התאמה → הסבר → Evaluator (עם לולאת חזרה)] → (בונוס: Gate/HITL → Code-Exec) → Output`

## 4. הרחבות
- **RAG**: מאגר ווקטורי קטן עם תמציות ממדריכי עיצוב (Apple / Google Noto / Samsung One UI) + מאגר נפרד לבדיקת דמיון לאימוג'ים קיימים (נוד 4).
- **Code-Exec** (בונוס): קוד Python היוצר SVG מהמפרט.

## 5. Evals — תכנון קונקרטי
1. **compare_models.py** — אותה משימת יצירת מפרט על מספר מודלים ב-OpenRouter; זמן אוטומטי; איכות/אמינות ב-LLM-as-judge לפי `rubric.md` (מודל שיפוט קבוע לכל ההשוואות).
2. **cost_eval.py** — עלות לכל שלב + כוללת, לפי tokens×תמחור; לפחות מודל בתשלום אחד; עלות תיאורטית למודלים חינמיים.
3. **scope_eval.py** — ≥5 שאלות מחוץ לתחום עיצוב אימוג'ים; מדד = refusal rate.

## 6. Roadmap מוסכם
1. בדיקת היתכנות מהירה — הרצת מודל הסיווג הקיים על כמה SVG לדוגמה (להחליט אם משלבים ב-Evaluator).
2. בחירת מודלים ב-OpenRouter לכל תפקיד (הבנה / ניסוח / שיפוט).
3. שלד הגרף (`graph.py`) — נודי ליבה בלבד (0–7 + Evaluator), בלי Code-Exec.
4. מאגרי RAG (מדריכי סגנון + אימוג'ים קיימים לבדיקת דמיון).
5. `cli.py` — Streaming, help, עלות — סביב תוצר המפרט בלבד.
6. שלושת ה-Evals + `rubric.md`.
7. חיבור LangSmith + בדיקות קצה-לקצה על שלב הליבה.
8. (בונוס) נוד Code-Exec + Gate/HITL + שילוב מודל הסיווג לפי תוצאת שלב 1.
9. אריזה: PyInstaller, `manifest.json`, `README.md`, `.env.example`, GitHub Releases.
